# Участие в разработке

Спасибо за интерес к Zentra CMF! Любой вклад приветствуется.

## Как начать

1. **Форкните** репозиторий на GitHub
2. Клонируйте свой форк локально:
   ```bash
   git clone https://github.com/ваш-username/zentra-cmf.git
   cd zentra-cmf
   ```
3. Добавьте upstream для синхронизации:
   ```bash
   git remote add upstream https://github.com/zentra-team/zentra-cmf.git
   ```

## Рабочий процесс

1. Синхронизируйтесь с основным репо перед началом работы:
   ```bash
   git fetch upstream
   git checkout main
   git merge upstream/main
   ```

2. Создайте ветку с понятным названием:
   ```
   fix/описание-бага
   feature/название-фичи
   docs/что-обновлено
   refactor/что-рефакторили
   ```

3. Вносите изменения, делайте коммиты:
   ```
   fix: краткое описание исправления
   feat: краткое описание новой функции
   docs: обновлена документация по X
   refactor: упрощена логика Y
   ```

4. Откройте **Pull Request в ветку `main`** вашего форка → основной репо.

## Требования к PR

- Один PR решает одну задачу
- Код проходит проверку синтаксиса (`php -l`)
- Нет отладочных вызовов (`dd()`, `var_dump()`, `console.log`)
- Изменения в публичном API или поведении сопровождаются обновлением документации

## Что принимается

- Исправления ошибок
- Улучшения производительности
- Новые типы полей (через `FieldInterface`)
- Улучшения документации
- Переводы

## Что не принимается без предварительного обсуждения

Крупные архитектурные изменения, новые зависимости, изменения публичного API — сначала откройте Issue или Discussions.

## Сообщить об ошибке

Откройте [Issue](https://github.com/zentra-team/zentra-cmf/issues/new/choose) по шаблону. Пожалуйста, не дублируйте существующие.

## Вопросы

Используйте [GitHub Discussions](https://github.com/zentra-team/zentra-cmf/discussions) — не Issues.
